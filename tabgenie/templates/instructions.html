In this task, you will annotate {{ annotation_example_cnt }} examples in total. For each example, you will see data on the left side and a text on the right side. Your task is to annotate errors in the text with respect to the data.

There are five types of errors that you can mark in the generated text:

  1. Incorrect fact: The fact in the text contradicts the data.
  2. Not checkable : The fact in the text cannot be checked given the data.
  3. Misleading: The fact in the text is misleading in the given context.
  4. Other : The text is problematic for another reason, e.g. grammatically or stylistically incorrect, irrelevant, or repetitive.

Use your mouse to highlight specific parts of the text containing the errors. To switch between error categories, repeatedly click on the highlighted text.

If the fact is probably true, but cannot be derived from the data, mark it as not checkable. If you are not really sure if the fact should be marked as an error, you can leave it unmarked. Always select the shortest span that contains the error, but include all the words related to the error.
